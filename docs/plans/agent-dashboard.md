# Plan: Agent Dashboard

## Goal
Give AGENT-role users a dead-simple dashboard that surfaces one opportunity at a time with a single clear next action, an AI-generated plain-English briefing, and an exact compliance checklist — no pipeline management, no filters, no noise.

## Success Criteria
- [ ] Agents who log in see the Agent Dashboard; ADMINs still see the existing pipeline dashboard
- [ ] The hero card shows: title, agency, deadline countdown, current stage, one action button
- [ ] Hero card includes an AI briefing section: narrative summary + talking points + compliance checklist
- [ ] Briefing is generated by GPT-4o on first view, cached to `opportunity.agentBriefing` — no re-call on reload
- [ ] "Generate Briefing" button shown if not yet generated; spinner during generation
- [ ] Below the hero, a compact list shows remaining active opportunities
- [ ] Each stage maps to one plain-English action label (no jargon)
- [ ] "No active opportunities" empty state is friendly and non-technical
- [ ] No TypeScript errors (`npx tsc --noEmit`)

---

## Role Routing
- `/dashboard` stays as the single route for all users
- `DashboardPage` checks `session.user.role`:
  - `ADMIN` → existing `<DashboardContent>` (pipeline + filters)
  - `AGENT` or `VIEWER` → new `<AgentDashboardContent>`

## Agent Briefing — AI Summary

### What it contains
Three sections, each short enough to read in under 60 seconds:

| Section | Content |
|---|---|
| **What they need** | 2–3 plain-English sentences describing what the government is buying and why it matters |
| **To win, you need** | Bullet list of exact qualification requirements: licenses, certifications, clearances, set-aside eligibility, experience thresholds |
| **Watch out for** | Bullet list of compliance requirements and gotchas: bonding, insurance minimums, mandatory on-site, reporting cadences, FAR clauses with teeth |

### Output JSON structure
```typescript
interface AgentBriefing {
  summary: string              // 2–3 sentence plain-English narrative
  talkingPoints: string[]      // 3–5 bullets: what makes this a good/bad fit
  qualifications: string[]     // Exact requirements to be eligible (licenses, certs, clearances, set-aside)
  complianceFlags: string[]    // Compliance requirements agents must acknowledge before bidding
  generatedAt: string          // ISO timestamp
}
```

### Generation logic
- Source: `opportunity.description` + `opportunity.rawData` + `opportunity.parsedAttachments` (if available)
- Model: GPT-4o
- Prompt emphasis: plain language, no jargon, no filler — every bullet must be specific and actionable
- Falls back to description-only if `parsedAttachments` is null
- Cached permanently in `opportunity.agentBriefing Json?` — only regenerated if user explicitly clicks "Refresh"

### New API route
`POST /api/opportunities/[id]/agent-briefing`
- Auth required (AGENT or ADMIN role)
- Reads `opportunity.description`, `rawData`, `parsedAttachments`
- Calls `generateAgentBriefing()` from `lib/openai.ts`
- Saves to `opportunity.agentBriefing`
- Returns `{ briefing: AgentBriefing }`

## Notes
- GPT-4o (not mini) — compliance bullets must be accurate, not fast
- Every bullet in `qualifications` and `complianceFlags` must be a specific, real requirement from the solicitation — no generic filler ("must comply with all applicable laws")
- If `parsedAttachments` is null, generate from `description` + `rawData` only — do not block
- `complianceFlags` empty array → omit the "Watch out for" section entirely
- Keep the agent view stateless — no search or filter state
- Do NOT show "Fetch from SAM.gov" to agents
