 Plan: Agent Experience — Dashboard + Bid Submission Workflow

## Goal
Give AGENT-role users a dead-simple dashboard that surfaces one opportunity at a time with a single clear next action and an AI-generated briefing. Agents can email the SOW to subcontractors, then submit their complete bid package to an admin for final review and approval before the bid goes to the government.

## Success Criteria
- [ ] Agents who log in see the Agent Dashboard; ADMINs still see the existing pipeline dashboard
- [ ] Hero card shows: agency, title, deadline countdown, current stage, one action button
- [ ] Hero card includes an AI briefing: plain-English narrative + qualification requirements + compliance flags
- [ ] Briefing is generated by GPT-4o on first view, cached to `opportunity.agentBriefing` — no re-call on reload
- [ ] "Generate Briefing" button shown if not yet generated; spinner during generation
- [ ] Below the hero, a compact list shows remaining active opportunities
- [ ] Agent can email the SOW (as PDF attachment) to subcontractors from the EmailDraftPanel
- [ ] Agent can submit the bid package with a note — triggers email to all ADMIN users
- [ ] A `BidApprovalRequest` record is created, visible in the admin review queue
- [ ] Admin can approve (→ bid status `REVIEWED`, stage `READY`) or reject with a comment
- [ ] Rejection sends agent an email with the admin's comment
- [ ] Approval sends agent an email confirming the bid is ready to submit
- [ ] No TypeScript errors (`npx tsc --noEmit`)

---

## Full Workflow

```
Agent logs in → sees Agent Dashboard (one opportunity at a time)
  ↓ Hero card shows: next step, deadline, AI briefing
  ↓ Agent works through stages in the workspace

When at BID_ASSEMBLY stage:
  ↓ (1) EmailDraftPanel → "Send SOW" button → emails SOW PDF to subcontractors
  ↓ (2) Subcontractors reply with quotes / agreements
  ↓ (3) Agent uploads those email responses as attachments
  ↓ (4) Agent clicks "Complete →" in BidEditorPanel
  ↓ (5) Submission package created → email sent to all ADMINs

Admin review queue (/admin/bid-approvals)
  ↓ (6) Admin opens package, reviews SOW + bid pricing + subcontractor responses
  ↓ (7) Admin approves or rejects with comment
  ↓ (8) Agent receives email notification of outcome
```

### New model: `BidApprovalRequest`
```prisma
model BidApprovalRequest {
  id            String   @id @default(cuid())
  opportunityId String
  bidId         String
  submittedById String
  agentNote     String?  @db.Text

  status        BidApprovalStatus @default(PENDING)
  reviewedById  String?
  reviewerNote  String?  @db.Text
  reviewedAt    DateTime?

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  opportunity   Opportunity @relation(fields: [opportunityId], references: [id], onDelete: Cascade)
  bid           Bid         @relation(fields: [bidId], references: [id], onDelete: Cascade)
  submittedBy   User        @relation("BidSubmitter", fields: [submittedById], references: [id])
  reviewedBy    User?       @relation("BidReviewer", fields: [reviewedById], references: [id])

  @@index([opportunityId])
  @@index([status])
  @@map("bid_approval_requests")
}

enum BidApprovalStatus {
  PENDING
  APPROVED
  REJECTED
}
```

## AI Briefing

### Output structure
```typescript
interface AgentBriefing {
  summary: string          // 2–3 sentence plain-English narrative
  talkingPoints: string[]  // 3–5 bullets: fit signals for this opportunity
  qualifications: string[] // Exact requirements: licenses, certs, clearances, set-aside
  complianceFlags: string[] // Compliance gotchas: bonding, insurance, FAR clauses
  generatedAt: string      // ISO timestamp
}
```

### Generation logic
- Source: `opportunity.description` + `opportunity.rawData` + `opportunity.parsedAttachments`
- Model: GPT-4o (accuracy over speed — compliance bullets must be real)
- Falls back to description + rawData if `parsedAttachments` is null
- Cached permanently in `opportunity.agentBriefing` — only regenerated on explicit "Refresh"
- Every bullet must be specific and actionable — no generic filler

## API Routes

### `POST /api/opportunities/[id]/agent-briefing`
- Auth: AGENT or ADMIN
- Reads opportunity description, rawData, parsedAttachments
- Calls `generateAgentBriefing()` from `lib/openai.ts`
- Saves result to `opportunity.agentBriefing`
- Returns `{ briefing: AgentBriefing }`

### `POST /api/opportunities/[id]/bid-approval`
Agent submits bid package for admin review.
- Auth: AGENT or ADMIN
- Body: `{ bidId: string, agentNote?: string }`
- Creates `BidApprovalRequest` with status `PENDING`
- Fetches all ADMIN users → sends email to each via `lib/email.ts`
  - Subject: `[Review Required] Bid Package — [opportunity title]`
  - Body: agent name, opportunity, deadline, recommended price, note, link to `/admin/bid-approvals/[id]`
- Returns `{ request: BidApprovalRequest }`

### `GET /api/admin/bid-approvals`
Admin queue — all pending + recent requests.
- Auth: ADMIN only
- Returns requests with `opportunity`, `bid`, `submittedBy` included
- Sorted by `createdAt desc`

### `PATCH /api/admin/bid-approvals/[id]`
Admin approves or rejects.
- Auth: ADMIN only
- Body: `{ action: 'APPROVE' | 'REJECT', reviewerNote?: string }`
- Updates `status`, `reviewedById`, `reviewerNote`, `reviewedAt`
- If APPROVE:
  - `Bid.status` → `REVIEWED`
  - `OpportunityProgress.currentStage` → `READY`
  - Emails agent: "Your bid package has been approved — ready to submit."
- If REJECT:
  - Bid status stays `DRAFT`
  - Emails agent with reviewer note: "Revision requested: [note]"
- Returns `{ request: BidApprovalRequest }`

## Notes
- GPT-4o (not mini) — compliance bullets must be accurate, not fast
- Every bullet in `qualifications` and `complianceFlags` must be a specific requirement from the solicitation — no generic filler
- `complianceFlags` empty array → omit the "Watch out for" section entirely
- Agent view is stateless — no search or filter
- Email to admins falls back silently if SMTP not configured (log warning)
- Multiple ADMINs all get notified; first to act wins (no double-approve guard needed initially)
- Agent can resubmit after rejection — creates a new `BidApprovalRequest`, old one stays in history
- SOW attachment in EmailDraftPanel pulls the most recent SOW with status `APPROVED` or `DRAFT` (whichever is latest)
- Admin review pages are accessible from `/admin` nav, not from the agent dashboard
- Do NOT show "Fetch from SAM.gov" to agents
